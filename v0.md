# Cicada v0: Bare Minimum Module Search

**Philosophy:** The simplest thing that could possibly work.

**Goal:** Search for an Elixir module and see its functions. Nothing more.

**Time:** 4-6 hours of focused work.

---

## Core Principle

> "Can I ask Claude Code 'What's in the User module?' and get an answer?"

If yes → v0 is complete.
If no → something is broken.

---

## What v0 Does

**Single Use Case:**
```
User: "What functions are in MyApp.User?"

Cicada: "MyApp.User has 5 functions:
  - authenticate/2 at line 42
  - create/1 at line 58
  - update/2 at line 73
  - delete/1 at line 89
  - get_by_email/1 at line 102"
```

**That's it.** Nothing else.

---

## What v0 Does NOT Do

- ❌ Find who calls what (call graph)
- ❌ Show git history
- ❌ Find tests
- ❌ Search documentation
- ❌ Show usage examples
- ❌ GitHub integration
- ❌ Error patterns
- ❌ Alternative functions
- ❌ Anything fancy

**Why?** Because those don't matter if module search doesn't work.

## Architecture

```
┌─────────────────────┐
│   Claude Code       │
└──────────┬──────────┘
           │ MCP
┌──────────▼──────────┐
│   mcp_server.py     │
│   • search_module   │ ← ONE TOOL
└──────────┬──────────┘
           │
┌──────────▼──────────┐
│   index.json        │
│   {                 │
│     "Module": {...} │
│   }                 │
└─────────────────────┘
```

**Data Flow:**
1. Run indexer once → creates `index.json`
2. Start MCP server → loads `index.json` into memory
3. Claude asks about module → search JSON → return results

**No database. No cache. No complexity.**

## Implementation Strategy

### Phase 1: Parser (1.5 hours)

**Goal:** Parse ONE Elixir file and extract modules + functions.

**File:** `parser.py` (~100 lines)

**What to extract:**
- Module name (from `defmodule`)
- Function name + arity (from `def` and `defp`)
- Line number
- Signature (first line of function)
- Visibility (def vs defp)

**What NOT to extract:**
- Function calls
- Dependencies
- Specs
- Documentation
- Anything else

**Test with:**
```elixir
defmodule Test do
  def hello(name) do
    "Hello #{name}"
  end

  defp private_func do
    :ok
  end
end
```

**Expected output:**
```json
{
  "module": "Test",
  "functions": [
    {
      "name": "hello",
      "arity": 1,
      "full_name": "hello/1",
      "line": 2,
      "signature": "def hello(name)",
      "type": "def"
    },
    {
      "name": "private_func",
      "arity": 0,
      "full_name": "private_func/0",
      "line": 6,
      "signature": "defp private_func",
      "type": "defp"
    }
  ]
}
```

**Success:** Can parse a single file correctly.

### Phase 2: Indexer (1.5 hours)

**Goal:** Walk repo, parse all `.ex` files, build index.

**File:** `indexer.py` (~80 lines)

**Index structure:**
```json
{
  "modules": {
    "MyApp.User": {
      "file": "lib/myapp/user.ex",
      "line": 1,
      "functions": [
        {
          "name": "authenticate",
          "arity": 2,
          "full_name": "authenticate/2",
          "line": 42,
          "signature": "def authenticate(email, password)",
          "type": "def"
        }
      ],
      "total_functions": 5,
      "public_functions": 3,
      "private_functions": 2
    }
  },
  "metadata": {
    "indexed_at": "2025-10-25T10:30:00",
    "total_modules": 45,
    "total_functions": 320,
    "repo_path": "/path/to/repo"

```

**What to index:**
- All `*.ex` files
- All `*.exs` files in `lib/` and `test/`

**What NOT to index:**
- `deps/`
- `_build/`
- `node_modules/`
- `.git/`

### Phase 3: MCP Server (2 hours)

**Goal:** Load index, serve ONE tool via MCP.

**File:** `mcp_server.py` (~120 lines)

**The Single Tool:**

```python
Tool(
    name="search_module",
    description=(
        "Search for an Elixir module by name and return all its functions. "
        "You can search by full name (MyApp.User) or partial name (User). "
        "Returns module location, all public and private functions with "
        "their signatures and line numbers."
    ),
    inputSchema={
        "type": "object",
        "properties": {
            "module_name": {
                "type": "string",
                "description": "Module name to search (e.g., 'MyApp.User' or just 'User')"
            }
        },
        "required": ["module_name"]
    }
)
```

**Implementation:**

```python
async def search_module(module_name: str) -> dict:
    """Search for a module and return its functions"""

    # Exact match
    if module_name in self.index['modules']:
        return self._format_module(module_name)

    # Partial match (search for "User" → find "MyApp.User")
    matches = []
    query_lower = module_name.lower()

    for full_module in self.index['modules'].keys():
        if query_lower in full_module.lower():
            matches.append(full_module)

    if len(matches) == 1:
        return self._format_module(matches[0])
    elif len(matches) > 1:
        return {
            "error": "Multiple modules found",
            "matches": matches,
            "hint": "Use full module name"
        }
    else:
        return {
            "error": "Module not found",
            "query": module_name,
            "hint": "Try a partial name or check spelling"
        }

def _format_module(self, module_name: str) -> dict:
    """Format module data for response"""
    data = self.index['modules'][module_name]

    return {
        "module": module_name,
        "file": data['file'],
        "line": data['line'],
        "functions": data['functions'],
        "summary": {
            "total": data['total_functions'],
            "public": data['public_functions'],
            "private": data['private_functions']
        }
    }
```

**Config:** `config.yaml`
```yaml
repository:
  path: /path/to/your/elixir/project

storage:
  index_path: ./data/index.json
```

**Usage:**
```bash
# Start server
python mcp_server.py

# Configure in Claude Code's MCP settings
# Then ask: "What's in the User module?"
```

**Success:** Can query modules from Claude Code.

## Dependencies

**requirements.txt:**
```
mcp>=1.0.0
tree-sitter>=0.20.0
tree-sitter-elixir>=0.1.0
pyyaml>=6.0.0
```

**Why these?**
- `mcp` - MCP protocol implementation
- `tree-sitter` - AST parser (robust, handles syntax correctly)
- `tree-sitter-elixir` - Elixir grammar
- `pyyaml` - Config file parsing

**Why NOT regex?** Tree-sitter handles edge cases (multiline, nested, macros).

## Troubleshooting

### Parser Fails
**Symptom:** Crashes on certain `.ex` files

**Fix:**
```python
# In indexer.py, add error handling
try:
    result = self.parser.parse_file(file_path)
except Exception as e:
    print(f"⚠️  Skipping {file_path}: {e}")
    continue
```

### Index is Empty
**Check:**
1. Repo path is correct: `ls /path/to/repo/lib`
2. `.ex` files exist: `find /path/to/repo -name "*.ex"`
3. Parser works: `python parser.py` (with test)

### MCP Server Won't Start
**Check:**
1. Config exists: `cat config.yaml`
2. Index exists: `cat data/index.json`
3. Dependencies installed: `pip list | grep mcp`
4. Python syntax: `python -m py_compile mcp_server.py`

### Claude Code Can't Connect
**Check:**
1. Absolute paths in MCP config (not relative)
2. Virtual environment activated in config
3. Server starts manually: `python mcp_server.py`
4. Check Claude Code logs

## Principles for v0 Development

### 1. One Thing at a Time
Don't write parser, indexer, and server together. Write parser. Test it. Then indexer. Test it. Then server.

### 2. Test with Real Data
Use a real Elixir project, not toy examples. Find a small open source project (~20 files).

### 3. Print Everything
```python
print(f"Parsing {file_path}...")
print(f"Found {len(functions)} functions")
print(f"Index has {len(index)} modules")
```
Remove prints later. Debug now.

### 4. Hard-code First
Don't make everything configurable. Hard-code paths. Hard-code limits. Make it work, then make it configurable.

### 5. Accept Imperfection
- Parser misses some edge cases? Fine.
- Index doesn't handle macros? Fine.
- Search isn't fuzzy? Fine.

Ship v0. Improve later.

### 6. Working > Perfect
```python
# This is ugly but works
if module_name in index:
    return index[module_name]

# Don't refactor to this until v0 works
return self.module_repository.find_by_name(
    module_name,
    search_strategy=FuzzySearchStrategy()
)
```

